App: Supplier Tracker
Initial Prompt: Software that manages relationships with suppliers and upcoming / historical purchase orders.  There needs to be a main dashboard page with lists of suppliers and upcoming orders, supplier details page with appropriate info (documents, upcoming / historical orders etc), order details page with appropriate info (details on what is being purchased, cost, documents etc).

Pages:

1. DashboardPage (/)
   Description: Main dashboard page for supplier relationship and purchase order management. Layout: sectioned view. Sections: (1) quick action buttons to add a supplier or order (2) Upcoming orders list: table or list of upcoming purchase orders with columns like order ID, supplier, expected delivery date, status, and total cost; each entry should link to the Order Details page. (3) Suppliers list: table or card list of all suppliers with key info such as supplier name, status, primary contact, and possibly a summary of open orders.
   Mockup: https://utfs.io/f/g4w5SXU7E8Kd5wMORO0tw0Ld4cHmvg86S39TjoGK5BDAsYF7

2. SupplierDetailsPage (/suppliers/:supplierId)
   Description: Supplier-specific details page. Layout: top section with supplier overview, followed sectioned content areas. Components: (1) Supplier information: supplier name, address, contact information, description, and other profile data. (2) Documents section: list or grid of documents related to the supplier (contracts, certifications, agreements). Each document item should show name, type, upload date, and actions such as view/download. (3) Comments added for notes about the supplier (4) Upcoming orders section: list or table of upcoming purchase orders associated with this supplier with details like order ID, expected delivery, status, and total cost; each order links to the Order Details page. (5) Historical orders section: list or table of past purchase orders with information such as order ID, delivery date, status (completed/cancelled), and final cost. Functionality: navigation back to the main dashboard, ability to open documents, drill down to order details, filtering between upcoming and historical orders, and basic search within the supplierâ€™s orders or documents.
   Mockup: https://utfs.io/f/g4w5SXU7E8KdCGl3wVHJin1BPSFeJzG2d6mrcMXaToqOAuZV

3. OrderDetailsPage (/orders/:orderId)
   Description: Purchase order details page. Layout: main content area with order summary at the top and detailed sections below. Components: (1) Order summary: order ID, supplier name (with link to Supplier Details), order date, expected delivery date, status, and overall cost. (2) Line items / what is being purchased: table listing each item with fields like item name/description, quantity, unit price, line total, and possibly SKU or part number. (3) Cost breakdown: subtotal, taxes, discounts, and total cost; possibly currency information. (4) Documents section: list of documents related to this purchase order (purchase order PDFs, invoices, shipping documents, etc.), each with attributes like file name, type, date, and actions to view/download. (5) Order history or activity log (if present): timestamps of order creation, approvals, status changes, and deliveries. Functionality: navigation back to dashboard and supplier details, ability to view/download related documents, and possibly editing or updating status depending on permissions.
